# 動作確認済みの環境とその考察

◯Win11（Core i5 13400F, 32GB memory, RTX3060）
◯MacOS Sequoia（Mac mini M4, 16GB memory）
◯Win10（AMD Ryzen 5 2400G, 8GB memory）
◯Zorin OS Lite（Core i5-4590, 8GB memory）
◯MX Linux（Raspberry Pi 5, 8GB memory）
　など
　
　開発は主に上2つで行っている。
　なお、MacOSはユーザーの方からTahoe 26.0.1での動作報告があった（MirrorShard v1.2.1）。

## ラズパイでの動作
　
・1.1.0を最後にバイナリの配布は停止しているが、一応ラズパイでも動く。さすがに起動はあまり速いとはいえないが、一度起動してしまえばマイクロSDカード起動のラズパイ５でも案外軽快に動作した。アイデアプロセッサ実装以降は未検証だが、少なくともエディタだけなら割と普通に使える。

・……ただ、OSがMX Linuxなら問題なく動作したのだが、Raspberry Pi OSでの動作が安定しなかったため、配布は中止した。

## Windowsでの動作

・そこそこのスペックがあれば動作自体は軽快（低スペック機でもわりと普通に動く）なのだが、メモリ32GBあっても起動には数秒かかる。

・とはいえやはりNVIDIA製のグラフィックボードがあるとローカルAI使用時には有利で、RTX3060程度であっても7Bとか12Bくらいの軽いモデルならGeminiと通信するよりよほど速い。
　本ソフトウェアのChain of Thought機能においてはAIにそれほどの性能は必要なく、7BくらいのAIでもそれなりに役に立つため、生成速度の早いローカルLLMには「思考の流れを中断させない」という十分なメリットがあると思う。

## Macでの動作

・本ソフトウェアともっとも相性がいいのは、おそらくMacである。
　Windowsでは32GBのメモリを積んでいても起動に数秒かかったのだが、Macはユニファイドメモリ16GBでも割とすぐに起動する。
　またWin11では一部のサブウィンドウでフリッカー対策のためtransparent = true;を指定しており、そのせいでウィンドウの角丸めができていないのだが、Macだけはなぜか不要だったので想定通りのデザインで運用できている。

・ただしMacはローカルLLMの運用にはあまり向いていないようで、AI使用時はGeminiを中心に使ったほうがいいかもしれない。

## Linuxでの動作

・Zorin OS LiteとMX Linux、Mint 21.1 Mate（Ubuntu 22.04LTSベース）あたりでは問題なく動作した。10年以上前の事務用PCに適当なLinuxを入れて再生したような代物でも普通に動くので、それなりに汎用性はありそうである。

・ただ、Ubuntu 14.04LTSベースの古いOSでは流石に動かなかった。おそらくglibcのバージョンが古すぎて動かないのではないか（と、Google Gemini 君は言っていた）。


# ChatGPTに対応しなかった理由

　技術的には可能だったのだが、私はOpenAIに課金していないのでChatGPTのAPIキーが取得できず、動作確認ができなかった。
　Geminiは無料でAPIキーが取得できるので、こちらはもちろん動作確認済みである。


# 開発経緯

　実は最初から創作専用エディタとして開発を開始したわけではなくて、「Macで使える巨大テキストファイル対応エディタがどうしても欲しかった」というのが本当の開発理由である。

　私は「数年分のゲームのプレイ記録」とか「20数年分の日記」とかを単一の階層つきテキストファイルで管理しているという変な人間なのだが、しかし数MB以上の巨大テキストが扱える（そしてアウトラインプロセッサとして運用できる）エディタというのは極めて数が限られる。WindowsはEmEditorがあるからいいが、問題はほかのOSである。
　特にMacが深刻で、フリーウェアの有名どころだとそれこそVS Codeくらいしか存在していない。――しかしそのVS Code、巨大ファイルを扱っていると、割と頻繁にクラッシュするようになってしまった（Zed editorも試してみたが同様）。
　ないなら自分で作るしか……ということで開発したのが本ソフトウェアというわけである。
　最初はMonaco Editorをベースに開発を始めたのだが、IMEの変換窓が変なところに出現してしまう仕様をどうやっても解消できず、結局CodeMirrorで作り直す羽目になった。

　まあそんなわけで、CodeMirrorの欠片から作られたのが本作MirrorShard、というわけだ。我ながら安直なネーミングである。

　そうして作られたMirrorShardだが、１つでもVS CodeやZed editorがクラッシュするような巨大ファイルを複数同時運用しても安定して扱えるのはさすがCodeMirrorといったところで、日々の常用エディタとして大変重宝している。

　そこから先は、自分が欲しい機能を手当たり次第に足していったらこうなった、というところである。
